# AI 漫剧统一生产产品需求说明书

## 1. 产品定位

LuminAI + Jellyfish 是面向工业化 AI 漫剧生产的“制片驾驶舱”，不是单点
文生视频工具。产品必须让用户从一句创意、一个梗概、一份小说文件或一个空
项目壳进入同一条可恢复、可审计、可批量生产的多集漫剧流水线。

核心价值：

- 把创意自动扩展为小说稿、分集脚本、分镜、资产圣经和生产任务。
- 保持角色、服装、道具、场景、镜头、风格和剧情状态连续。
- 用拍摄门禁、自动 QA、失败重试和后期计划降低批量产出返工。
- 让制片、编剧、美术、分镜、导演和运营都能在同一项目状态里协作。

## 2. 目标用户与场景

| 用户角色 | 主要任务 | 关键需求 |
| --- | --- | --- |
| 制片人 | 从创意启动多集项目，判断能否进入批量生产 | 统一创建入口、模块进度、门禁阻塞、成本可控 |
| 编剧/故事编辑 | 检查小说稿、分集节奏、上下集钩子 | 可回到章节/剧本模块编辑，不丢生产状态 |
| 分镜导演 | 审核镜头顺序、景别、机位、动作拍点 | Shot Graph、镜头准备度、可追踪补缺 |
| 美术/角色设计 | 稳定角色外观、服装、道具、场景、参考图 | 资产圣经、网络参考采集、授权核查说明 |
| 生成运营 | 创建图片/视频任务、QA、重试、成片导出 | 运行时隔离、任务中心、失败镜头局部重试 |

## 3. 行业痛点

| 行业痛点 | 现象 | 产品解法 |
| --- | --- | --- |
| 入口割裂 | 空项目、文本生成、Film Core 多个按钮让新用户不知道从哪开始 | 顶层统一为 `创建 AI 漫剧`，内部选择自动生成或空项目 |
| 一句话难落地 | 创意无法稳定变成多集剧本、分镜和资产 | `text-to-drama` 自动生成小说稿、分集、镜头和资产圣经 |
| 角色漂移 | 多集、多镜头里脸、服装、道具不一致 | 角色注册、身份参考、服装/道具/场景绑定、QA 检查 |
| 分镜状态不可见 | 分镜任务排队时看起来卡在 0% | 创建任务后立即显示非零进度，并持续轮询任务状态 |
| 参考素材不可控 | 网络图和视频素材来源不明，版权风险高 | 只生成搜索词和候选 URL，要求人工筛选授权素材 |
| 批量重试成本高 | 一个镜头失败导致整集返工 | QA 定位失败镜头，Retry 只重生成失败片段 |
| 成片链路断裂 | 生成视频后还要人工整理字幕、TTS、转场 | Film Core 给出后期计划和最终导出模块 |

## 4. 产品总流程

```text
创建 AI 漫剧
→ 选择自动生成 / 空项目
→ 输入一句话、梗概、正文或上传 .txt/.md 小说
→ 生成小说稿
→ 拆分多集脚本
→ 抽取分镜 / Shot Graph
→ 生成角色、演员身份参考、服装、道具、场景、VFX 资产圣经
→ 生成角色图片/视频网络参考采集任务
→ 进入 Film Core
→ 检查 AI 漫剧生产进度和拍摄门禁
→ 创建图片/视频生产任务
→ 自动 QA
→ 局部 Retry
→ 后期合成与多集导出
```

## 5. 创建入口需求

项目列表只保留一个顶层入口：`创建 AI 漫剧`。

弹窗内有两个模式：

| 模式 | 必填 | 输出 | 进入页面 |
| --- | --- | --- | --- |
| 自动生成漫剧 | 原始创意/梗概/正文，或上传 `.txt` / `.md` 后自动填充 | Project、Chapter、Shot、角色/服装/道具/场景、VFX、工作流状态、参考采集任务 | `/projects/{projectId}?tab=filmCore` |
| 空项目 | 项目名称 | 只有 Jellyfish 项目壳，用于已有剧本或手工导入资产的团队 | `/projects/{projectId}?tab=chapters` |

创建参数：

- `project_name`：项目名称，自动生成模式可为空，后端从文本提取默认名。
- `source_text`：一句话、梗概、正文或小说文件内容。
- `style` / `visual_style`：项目类型和视觉风格。
- `episode_count`：目标集数。
- `shots_per_episode`：每集目标镜头数。
- `default_video_ratio`：默认画幅，竖屏漫剧默认 `9:16`。
- `automation_mode`：自动推进或人工门禁。
- `reference_harvest_enabled`：是否生成角色网络参考采集任务。

## 6. Film Core 功能模块

Film Core 必须展示 `AI漫剧生产进度`，它和底层 runtime pipeline 分离：
runtime pipeline 表示系统执行层，production modules 表示制片人该做什么。

| 模块 | 完成标准 | UI 操作 |
| --- | --- | --- |
| 项目创建 | 项目、风格、画幅、seed 已存在 | 返回项目设置 |
| 小说 / 剧本 | 章节有原文或精简文本 | 返回章节编辑 |
| 分集拆解 | 已生成章节/集数和每集摘要 | 返回章节列表 |
| 角色 / 服装 / 道具 / 场景 / 视效 | 资产圣经和绑定关系存在 | 返回角色/资产模块 |
| 角色网络图片 / 视频参考 | 生成搜索词、候选 URL、授权核查说明 | 在 Film Core 查看并人工筛选 |
| 分镜 / Shot Graph | 分镜和镜头细节存在 | 返回章节分镜 |
| 镜头准备 | 至少有 ready 镜头，缺口明确 | 返回镜头准备页 |
| 图片 / 视频生成 | 拍摄门禁通过后可创建生产任务 | Film Core 创建任务 |
| 自动 QA / Retry | 有视频产物后可评分和生成失败补丁 | Film Core 查看 QA/Retry |
| 后期 / 成片输出 | 通过 QA 的片段进入字幕、TTS、转场、导出计划 | 返回后期模块 |

每个模块字段：

- `key`：稳定模块键。
- `title`：模块标题。
- `status`：`waiting` / `active` / `blocked` / `done`。
- `progress`：0-100 的结构化进度。
- `summary`：模块当前摘要。
- `tasks`：模块检查项。
- `next_action`：下一步动作。
- `route_hint`：返回编辑页面的路径。
- `can_return`：是否允许返回该模块。
- `blockers`：阻塞原因列表。

## 7. 百炼大模型配置需求

本地重启后，后端必须能从仓库根目录 `.env` 或后端 `.env` 读取百炼密钥。
支持变量优先级：

1. `ALIYUN_BAILIAN_API_KEY`
2. `BAILIAN_API_KEY`
3. `DASHSCOPE_API_KEY`
4. `VITE_API_KEY`（兼容当前已有根 `.env`）

默认文本模型使用阿里百炼兼容 OpenAI 的接口：

- Provider ID：`aliyun_bailian`
- Provider 名称：`阿里百炼`
- Base URL：`https://dashscope.aliyuncs.com/compatible-mode/v1`
- 默认模型：`qwen-plus`，可用 `ALIYUN_BAILIAN_MODEL`、`BAILIAN_MODEL`
  或 `DASHSCOPE_MODEL` 覆盖。

运行时不得在接口响应、日志或文档中回显密钥明文，只能展示是否已配置。

## 8. 独立端口需求

本项目本地开发端口必须避开常见端口和其他项目：

- Jellyfish 后端：`http://127.0.0.1:24731`
- Jellyfish 前端：`http://localhost:24732/projects`
- LuminAI 轻量运行时仍可按需使用 `8765`，但 AI 漫剧主工作台以上述
  Jellyfish 端口为准。

启动脚本需要支持环境变量覆盖：

- `BACKEND_PORT`
- `FRONT_PORT`
- `BACKEND_URL`
- `BACKEND_HOST`
- `FRONT_HOST`

## 9. 非功能需求

- 稳定性：创建、分镜、生产任务必须可重试、可恢复。
- 可扩展性：大模型、图片模型、视频模型都通过 Provider/Model adapter 隔离。
- 可维护性：业务逻辑放 service，route 保持薄层。
- 可审计性：关键自动化动作写入任务账本。
- 安全性：密钥不回显，网络参考素材不自动下载。
- 用户体验：入口少、状态清楚、阻塞原因可理解、可返回编辑。

## 10. 验收标准

- 项目列表只有一个顶层新建入口 `创建 AI 漫剧`。
- 一句话可生成项目、小说稿、分集、分镜、角色、服装、道具、场景、
  VFX、参考采集任务和 CineForge 工作流状态。
- `.txt` / `.md` 小说文件可在浏览器本地读取并填入同一创建流程。
- Film Core 展示生产模块进度、阻塞项、下一步动作和返回路径。
- Film Core 展示角色图片/视频参考搜索词、候选搜索 URL 和版权策略。
- 分镜任务创建后前端可见非零进度。
- 后端默认端口为 `24731`，前端默认端口为 `24732`，CORS 和生成客户端一致。
- 百炼 Provider/Model 可从 `.env` 自动引导为默认文本大模型，且不泄露密钥。
- 自动化测试、手工 smoke、文档、用户手册和任务进度索引全部更新。
